The PowerShell Conference Asia 2016

2 conference days and 1 day of pre-conference workshops on PowerShell and DevOps, delivered by speakers from around the world and less than 150 available seats.

When

21st - 22nd October (Pre-Conference Day 20th October)

Where

Singapore
Microsoft

For three days,
PowerShell experts deliver some great content.

Join us for the second PowerShell Conference Asia, held in Singapore, where PowerShell speakers from Asia and around the world come together to bring attendees in-depth PowerShell and DevOps content.

Speakers include several members of the Windows PowerShell team from Microsoft headquarters in Redmond and a strong line-up of MVPs, well-known international speakers, and community contributors. They'll cover in-depth topics on the PowerShell language and how you can use PowerShell to automate the technologies you use every day. There will be strong focus on using PowerShell to enable DevOps practices whether On Premises or in the cloud.

The main event runs on Friday and Saturday, but we also have a pre-conference day on Thursday, where we will have some vendor-led workshops where you will have the oppotunity to be introduced to and get hands on with tools and technologies in the DevOps and Automation space. If that's not your thing, we have a 1 day PowerShell refresher crash course led by one of the best PowerShell trainers in the United States. All included in your ticket price. This is geared up to make sure you get the most from being a conference attendee.

Our full schedule on Saturday ends with a closing session of drinks and nibbles at a local bar. If you have travelled to Singapore, you can head right home on Sunday or some sightseeing and a follow up lunch with peers and speakers.

This event is made possible by several well-known sponsors who will be announced very soon. Stay tuned for our speakers and schedule announcements also. Last year was awesome, this year will be even better.

Please register using the mailing list link to be notified when the tickets go on sale.

10:00 - 11:00

Building PowerShell GUI Tool Solutions

Dr. Ferdinand Rios
SAPIEN Technologies

PowerShell is great for automating processes, managing servers and just about anything else you need to do on the Windows platform from an administrative perspective. As an admin though, you may need to service individuals who know nothing about PowerShell, command lines or consoles. You will need advanced GUI tools that you or your customers can run on their desktops to get you the information you need to solve their problems. Additionally, advanced PowerShell developers and administrators need more advanced tooling. Using PowerShell Studio, we will walk you through the workflow of creating GUI based tools that run PowerShell behind the scenes.

11:00 - 12:00

Windows Desired State Configuration

Benjamin Hodge
Kemp Technologies

Are you paying attention to DevOps? Adoption of DevOps practices can greatly improve your company’s deployment efficiency. PowerShell Desired State Configuration (DSC) helps teams take the management of their Windows-based infrastructure into the DevOps space by capturing their infrastructure as "code". The declarative PowerShell model enables autonomous, and transparent configuration and deployment of Windows infrastructure and components. Capturing infrastructure as "code" is not only a means to manage what they have, at scale and speed, it is also a way to decouple the complexity of their existing environment in order to facilitate a migration to the cloud. Come see how DSC works and how you can use it to make configuration of internal applications, databases, and other infrastructure elements more efficient.

Lunch

Infrastructure as Code with Desired State Configuration (DSC)

Ravikanth Chaganti
Microsoft MVP

With DevOps pratices gaining momentum in the enterprises and startups, it is important for infrastructure administrators gear up for this new way of working. Infrastructure as code practices enable administrators to implement infrastrcture configuration as code. Administrators in their new role as infrastructure developers need to think and act like application developers. This sessions the attendees through what infrastructure as code is and what role DSC plays here.

10:00 - 11:00

Creating Elegant Code

Benjamin Hodge
Kemp Technologies

In the rough and tumble world of IT Pro’s and project deadlines there’s a lot of incentive to just bang out a few “one liners” and focus on “getting the job done”. At the time this makes perfect sense but there’s a big price paid in terms of re-usability and maintenance of that code if we just leave it that way after the dust has settled. In this talk Benjamin will show you how to use a few simple rules to quickly take the ugliest code that you (or others!) have created in the heat of the moment and produce beautiful, elegant functions and modules that you can be proud of.

11:00 - 12:00

PowerShell Security Features and Threat Management

Jaap Brasser
Technical Consultant

In the PowerShell security features and threat management session I will walk you through a number of considerations of PowerShell and security. This will include the new features such as Deep script block logging, PowerShell integration with Antimalware Scan but also how PowerShell can be used as a post exploitation tool and I will give some insights on how to defend yourself against such attacks.

12:00

Lunch

13:00 - 14:00

Object Oriented .NET with PowerShell

Punit Ganshani
Microsoft MVP

Object Oriented Programming has been an essential skill for developers and over the years the technical fraternity has witnessed several OOP languages and leveraged their benefits. Using object orientation in PowerShell can take your scripts go to different level. The session will focus on using .NET with PowerShell – creating classes in PowerShell, objects, reference types, events, loading assembly and alike. Leverage your .NET skills and create managed-PowerShell scripts!

14:00 - 15:00

Why to Pester PowerShell Scripts?

Deepak Dhami
Microsoft MVP

As we write more and more Production PowerShell code, good quality testing is essential to ensure it is Production ready. PowerShell now includes a Testing Framework for PowerShell; Pester. In this session, we will take an already written function and write Pester tests for it. Also we will take a look at writing tests first and then do Behaviour driven development of the code. Build Trust into your code before you deploy and use in Production!

15:00

Break

15:30 - 16:30

Manage your server securely using PowerShell Web Access and custom constrained endpoints

Ritesh Modi
Microsoft

In the current world of Information Technology, protective measures do not stop at the network edge. There have been reports to indicate the need to protect assets using measures that reduce administrative access. While the principle of least privilege has always been known to IT Security professionals, there is a need in the industry for a standardized method of constructing an operator experience that reduces access with a more sophisticated level of granularity than what is available in many traditional access control models. Custom Remoting Constrained endpoints helps us to achieve that. PowerShell Web Access (PSWA) feature in Windows Server 2012 allows remote administrative access to a set of servers with granular delegated access and only expose specific administrative privileges to different levels to support teams. PowerShell Web Access combined with custom constrained endpoints bring you a great way to manage and delegate admin access remotely. This level 400 demo heavy session will delve into the details about using Powershell Web Access along with Remoting constrained endpoints.

16:30 - 17:30

Configuration and Management of Dell Computers Client/Desktop using PowerShell

Girish Prakash
Dell Inc.

A Windows PowerShell provider allows any data store to be exposed like a file system as if it were a mounted drive. In this session I will share my experience with writing PowerShell Providers for systems, the challenges I came across while doing so, and how I solved them. You will learn when you should consider creating a Provider versus cmdlets, and how to integrate help into your Providers you create along with the generic commands used. This session will feature some great demos to demonstrate the power of Providers using DELL PowerShell Provider and how easy it is going to get managing Dell systems (primarily client systems). As a natural next step to management using PowerShell, I will briefly demonstrate how DSC can be wrapped around the existing implementation.

17:30
